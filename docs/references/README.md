# References

Pinned external reading for agents: library documentation dumps (`*-llms.txt`),
API references, and spec excerpts that should travel with the repo so an agent
never has to guess at an upstream API.

**Conventions**
- One file per source, named `<topic>-llms.txt` or `<topic>.md`.
- Note the source URL and the date/version captured at the top of each file.
- Keep these read-only; they are inputs, not working docs.

_Empty until a phase needs a pinned reference (e.g. NeMo Agent Toolkit or
Guardrails API dumps)._
